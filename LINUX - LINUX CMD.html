CRONTAB

################/var/spool/cron/crontabs/############################

agendar execução de comandos

min    hour       DOM    Mon    DOW    CMD

min - minuto 0 a 59
hour - 0 a 23
DOM - dia do mês - 1 a 28/29 ou 30/31
Mon - 1 - 12 - janeiro a dezembro
DOW - dia da semana - 0 a 6 - domingo( 0 ) a sábado ( 6 )
CMD - comando a executar

*-> o intervalo todo possível ex: no min é 0 a 59  | no hour é 0 a 23....
x-y -> de x a y \ ex: 0-25 ( de 0 a 25  )
x,y -> no x e no y \ ex:0,25 ( no 0 e no 25)
x/y -> de y em y no intervalo x  \ ex: 0-40/10 ( de 10 em 10 do intervalo 0 a 40 )

convém não usar o DOM e o DOW pois é demasiado ambíguo

RSYNC

apt install tsync - instala o rsync

upload de um file
    rsync "opções" "file a ser mandado" "utilizador"@"ip":"folder no destino"
download de um file
    rsync "opções" "utilizador"@"ip":"folder no destino" "folder para guardar o file"

Opções
    -v : Verbose - Descritivo  
    -r : copia dados recursivamente (mas não preserva data/hora e permissões) 
    -a : archive mode - modos de arquivo permite copiar ficheiros recursivamente e também preserva links simbólicos, permissões de ficheiros, data/hora e utilizador e grupos. 
    -z : compress file data – Comprime os dados 
    -h : human-readable - mostra o tamanho dos ficheiros no formato facilmente reconhecido (KB,MB, GB)

Exemplos
Copiar/sincronizar um ficheiro na mesma máquina:
    rsync -zvh backup.tar /tmp/backups/ 
Copiar/sincronizar um directório na mesma máquina: 
    rsync -avzh /root/rpmpkgs /tmp/backups/ 
Copiar/sincronizar um directório local para uma máquina remota:
    rsync -avz rpmpkgs/ root@192.168.27.101:/home/
Copiar/sincronizar um directório local numa máquina remota para uma pasta local: 
    rsync -avz root@192.168.27.101:/home/rpmpkgs /tmp/rpmpkgs/ 
Copiar/sincronizar um directório local para uma máquina remota, indicando a porta do SSH aberta: 
    rsync -avz -e “ssh –p901” rpmpkgs/ root@192.168.27.101:/home/ 
Mostrar o progresso enquanto transfere os dados: 
    rsync -avzhe ssh --progress /home/rpmpkgs root@192.168.0.100:/root/rpmpkgs


Para aceder via ssh sem colocar password
    ssh-keygen –t rsa - Cria uma chave ( Após inserir este comando deixe a password em branco e não é necessário especificar o ficheiro. )
    ssh-copy-id -i .ssh/id_rsa.pub "root"@"ip"

/etc/hosts.allow -   lista  de regras para ver quem pode aceder ao serviço
    serviço: ip/rede/host/dominios/wildcards
    exs:
        ssh: 182.168.1.1 - o host com o ip 192.168.1.1 vai ser permitida a entrada
        ssh: 182.168.1. - o host que tenha no ip 192.168.1. vai ser permitida a entrada
        vsftpd:  10.4.0.0/255.255.0.0 - ip com mascara
        vsftpd:  10.4.0.0/16 - ip com mascara
        
        ssh:  hairday.insecure.net - o host com este dominio vai poder entrar
        ssh:  .insecure.net - alguem que tenha isto no dominio vai poder entrar ex: cxxxx.xxcx.insecure.net

        portmap: ALL -  todos os clientes independentemente do nome, IP ou domínio
        portmap: PARANOID - Clientes cujo nome não coincide com o IP (falsificação de nome)
        portmap: LOCAL -  Cliente que tem origem na mesma máquina do servidor ou está sob o mesmo domínio.
        portmap: UNKNOWN - Cliente cujo nome e IP não consegue ser resolvido.
        portmap: KNOWN -  Cliente cujo nome e IP pode ser resolvido.  

        ssh: ALL EXCEPT x.x.x.x - Todos menos os com o ip x.x.x.x

/etc/hosts.deny-   lista  de regras para ver quem não pode aceder ao serviço
    serviço: ip/rede/host/dominios/wildcards
    exs:
        ALL:ALL - nega tugo que não esteja no allow

        Este ficheiro é corrido depois do allow, ou seja se no allow permite algo e neste nega algo, ele passa á mesma
        Por defeito, se algum ip ou dominio não estár negado, ele passa ex: se no allow permitir o x.x.x.x e no deny não por  all:all o y.y.y.y passa
